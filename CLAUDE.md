## 근거 기반 제안 원칙 (CRITICAL — 모든 작업에 적용)

### 태그 규칙

모든 기술적 제안, 아키텍처 결정, 도구 사용 방법은 근거 유형을 **반드시 명시**해야 한다.

- `[공식]` — 공식 문서/블로그/GitHub README에 명확히 서술된 사실 + URL 또는 문서명 필수
- `[외부]` — 논문, 외부 블로그, RFC 등 비공식 출처 + URL 필수
- `[추측]` — 공식 문서 없음, 경험적 판단 또는 불확실한 사항

**금지**: 태그 없이 기술적 사실처럼 서술 ("~이다", "~해야 한다" 등 출처 불명)

### [추측] 알람 의무 (MANDATORY)

`[추측]` 태그를 사용할 때마다 반드시 다음 형식의 알람 블록을 응답에 포함해야 한다.

```
⚠️ [추측 알람] 확인 필요
- 항목: <추측으로 서술한 내용 요약>
- 이유: <왜 공식 문서를 찾지 못했는지>
- 확인 방법: <공식 문서 URL 또는 확인 방법 제안>
- 조치: 확인 후 [공식] 또는 [외부]로 태그 업데이트 필요
```

**알람 생략 금지**: `[추측]`이 포함된 응답에서 위 블록 누락 시 규칙 위반.
**복수 추측**: 여러 항목이 `[추측]`이면 각각 별도 알람 블록 생성.

### Spec 문서 인용 규칙

`specs/ai-agent-engineering-spec-2026.md` 요구사항 참조 시:
- Spec 내 Anthropic 공식 블로그/Claude Code Guide 링크가 있으면 → `[공식]`
- Spec 내 참조 링크가 없거나 추론된 내용이면 → `[추측]` + 알람 블록 필수
- 김영동 세미나 등 외부 발표 자료 → `[외부]`

## 필수 규칙 (CRITICAL — 모든 작업에 적용)

### 파이프라인 실행 판단 기준

**복잡한 작업 → 파이프라인 필수:**
- 다중 산출물 생성 (3개 이상)
- 데이터 검증이 필요한 작업
- 계약서 초안 생성, 법적 검토가 필요한 문서

**단순 작업 → 직접 실행 허용:**
- 텍스트/수치 수정
- 파일 순서 변경, 삭제, 병합, 백업
- 사용자가 "직접실행"을 명시한 경우 — 범위 무관하게 직접 실행

**파이프라인 실행 규칙:**
- **Executor/Reviewer는 step 정보만 수신** — 전체 Sprint_Contract 전달 금지
- **병렬→순차 전환 금지** — 사용자 동의 없이 변경 금지
- **MCP 불가 항목 즉시 보고** — 무단 대체 금지

### 파일 관리 규칙

- 원본 파일 외에 백업/수정본을 자동 생성하지 않는다
- 모든 작업 중간 결과는 `.pipeline/` 폴더에만 저장한다
- `results/` 폴더에는 최종 산출물만 유지한다
